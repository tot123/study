## 为什么会出现「Skill」这个概念？

这里说的 **Skill / Skills**，不是一般意义上的“技能”，而是近一两年在大模型、Agent 圈里非常火的那个 **Skill 框架 / Skill 文件夹** 概念，最早由 Anthropic 在 2025‑10‑16 随 Claude 推出 Claude Skills / Agent Skills 引爆。

### 1. 核心背景：大模型从“聊天机器人”进化到“数字员工”

通用大模型（ChatGPT、Claude 这类）本身很聪明、会聊天，但存在几个痛点：

- 会“说”，但不会稳定、标准地“做事”
- 每次要靠人写一大堆 prompt 教它流程
- 不同平台上调教好的“能力”没法直接迁移
- 容易因为对话风格变化、上下文太长而表现不稳定

**Skill 的出现，本质上就是为了让 AI 从「聊天机器人」升级成「带具体专业技能的数字员工」。**  
也就是：不只是会回答，而是能“按一套标准流程完成具体工作”。

### 2. Skill 具体解决了哪些问题？

综合各篇技术解析和产品文章的结论，可以概括为四类核心问题：

1. **上下文爆炸：Prompt 越写越长，反而越笨**

   - 旧做法：在 System Prompt 里塞进几千字说明、N 个工具描述，模型每次都要“全看一遍”，Token 成本高、注意力涣散、容易漏步骤。
   - Skill 的做法：  
     - 先有一个“技能目录”（Skill 列表），  
     - 需要哪个技能，**按需加载**对应 Skill 文件夹里的说明和资源。  
   → 相当于：让模型“先知道你会这些活”，真正要干某个活时再详细看“操作手册”，减轻上下文压力。

2. **有手没脑：Tools 只有“手”，没有“怎么干的经验”**

   - 旧做法：Tools / Functions 只是“能调一个接口”，但没有：
     - 先做什么、后做什么的流程
     - 风险点 / 合规要求
     - 经验性的判断和边界
   - Skill 的做法：  
     - 把 **流程（SOP）+ 工具调用逻辑 + 参考资料** 打包成一个 Skill 文件夹  
     - 告诉 AI：**第一步干什么、第二步干什么、什么时候用哪个工具、要注意什么**
   → 本质：从“给你工具”升级到“给你一整套可复用的专业方法论”。

3. **跨平台割裂：在一个平台调教半天，换个平台全没了**

   - 现实问题：  
     - 你在某个 AI 产品里调了一套很顺手的工作方式，换成别的平台，得从头再教。
   - Skill 的思路：  
     - 用 **标准化的文件夹结构 + 通用格式（如 SKILL.md + 资源文件）** 表达一个技能  
     - 只要不同产品都支持这个标准，就能像装 App 一样，把同一个 Skill 在不同 AI 助手、不同 IDE、不同 Agent 系统里复用[1][3]。
   → 这是把“能力从平台里剥离出来”，变成 **用户自己的数字资产**。

4. **幻觉与不稳定：光靠 prompt，执行经常“跑偏”**

   - 旧状况：  
     - 今天回复这样，明天可能说法又变；  
     - 对话多轮一长，模型容易被后面的话“带跑”，忘了初始约定。
   - Skill 的做法：  
     - 每个 Skill 是一个 **有明确输入输出格式和边界的“沙盒”**  
     - 执行该 Skill 时，逻辑更固定、可预测，更适合严肃商业场景（报表、合规文档、财务流程等）。
   → 核心：用模块化、标准化的“技能包”约束 AI 的行为，让它“少发挥，多照规矩办事”。

### 3. 从工程角度看：Skill 是什么？

综合多篇技术说明，Skill 的本质可以这样理解：

> **Skill = 带脚本和资料的文件夹 + 一份写清楚“怎么做”的说明书（SKILL.md）**

典型内容包括：

- `SKILL.md`：  
  - 元信息：name, description, 适用场景、输入输出格式  
  - Instructions：具体步骤 / 原则 / 注意事项  
  - Examples：示例输入输出
- `examples.md`、`reference.md`：案例和参考文档
- 可选的脚本、API 定义等：作为“执行的肌肉”，提供确定性逻辑

所以，在 AI 领域里会出现 Skill 概念，是因为：

> **大家发现，真正限制 AI 的，不是“智商”，而是缺乏可复用、可迁移的「流程性知识」和「经验打包方式」。Skill 就是为了解决这个问题的工程化载体。**
