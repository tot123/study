# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 仓库性质

这是一个**个人技术知识库**（笔记），主要包含 AI 工具、开发工具、技术实践的中文文档。不是可执行的软件项目。

### 内容主题

- **Claude Code**: 完整使用指南、命令系统、Skills 系统、MCP 服务器、最佳实践
- **AI 工具**: Docker 安装 Ollama、第三方模型集成、智能路由工具
- **开发工具**: 终端工具、自动化工具、插件系统

### 文档组织

文档采用数字编号前缀（`00001-`, `00200-`, `00201-` 等）分类：

- `00001-*`: 基础工具安装和配置
- `00200-*`: Claude Code 核心功能指南
- `00201-*`: Claude Code 扩展系统（命令、竞品分析）
- `00202-*`: MCP 服务器集成
- `00203-*`: Skills 系统
- `00204-*`: 第三方模型集成
- `00205-*`: 智能路由
- `00206-*`: 最佳实践

## 文档维护原则

### 内容更新

1. **保持同步**: 当 Claude Code 发布新版本或功能变更时，及时更新相关文档
2. **实战验证**: 添加新的使用经验或案例前，先验证有效性
3. **版本标记**: 在文档顶部标注最后更新日期和相关版本

## 新文档创建指南

### 选题原则

- 补充现有知识库的空白领域
- 记录实际使用中的问题和解决方案
- 整合多个文档的关联内容

### 命名规范

```
[编号]-[主题]-[子主题].md

示例:
00207-claudecode-插件开发.md

```

### 内容来源

可以通过以下方式获取内容：

1. **官方文档**: 使用 `mcp__web-reader__webReader` 抓取官方文档
2. **实际测试**: 亲自验证功能并记录步骤
3. **社区资源**: 参考 GitHub、博客等社区内容（注明来源）
4. **经验总结**: 记录使用中的技巧和坑点

```

```

适用于：
- 更新现有文档到最新版本

- 研究新功能和最佳实践

## 常见任务

### 更新现有文档

1. 使用 Read 工具查看当前内容
2. 使用 Web Reader 获取最新信息
3. 对比差异并更新
4. 在文档顶部添加更新日期

### 创建主题文档

1. 确定编号（避免冲突）
2. 编写大纲和目录
3. 填充内容（优先使用官方来源）
4. 添加代码示例和配置示例
5. 交叉引用相关文档

### 文档格式化

- 使用中文标点（全角符号）
- 技术术语保持英文原文
- 代码块使用语法高亮
- 添加必要的 emoji 图标增强可读性
